# 键盘宏播放器 — 软件策划书（SOP）

**文档版本：** v1.0  
**日期：** 2026-03-05  
**技术栈：** .NET 8 / WinAPI  

---

## 一、项目概述

本项目旨在开发一款轻量级**键盘宏播放器**，支持用户通过全局热键触发自定义按键序列（宏），实现按键自动化操作。软件以单 EXE 形式发布，无需安装，常驻系统托盘运行。

---

## 二、核心模块划分

系统共拆分为 **4 个核心模块**：

| 模块 | 职责 |
|------|------|
| 宏规则模块 | 定义、解析按键序列 |
| 热键监听模块 | 全局监听触发键 |
| 宏播放器模块 | 按序执行模拟按键 |
| 配置存储模块 | 持久化宏配置 |

---

## 三、整体运行流程

```
启动程序
  ↓
读取配置文件（JSON）
  ↓
注册全局热键
  ↓
后台监听按键事件
  ↓
触发热键 → 匹配宏规则
  ↓
宏播放器按序执行
  ↓
结束 / 等待下次触发
```

---

## 四、模块详细设计

### 4.1 宏规则模块

**职责：** 定义宏的数据结构，并将字符串格式的宏解析为可执行序列。

**宏的基本单元：**

| 字段 | 类型 | 说明 |
|------|------|------|
| Key | string | 按键名称，如 `D`、`Q`、`F1` |
| DelayMs | int | 按键后等待时间（毫秒） |
| IsLongPress | bool | 是否为长按 |

**解析示例：**

输入字符串 `DDQQ`，解析结果：

```
D → delay 50ms
D → delay 50ms
Q → delay 50ms
Q → delay 50ms
```

**SOP 步骤：**
1. 接收热键绑定的宏字符串
2. 逐字符拆分为按键单元列表
3. 为每个单元附加默认延迟（默认 50ms）
4. 输出结构化按键序列，传递给播放器模块

---

### 4.2 热键监听模块

**职责：** 全局监听用户设定的触发键，与宏规则进行映射。

**触发映射示例：**

```
F4  →  宏: DDQQ
F6  →  宏: 123
F8  →  宏: WASD
```

**SOP 步骤：**
1. 启动时从配置加载热键-宏映射表
2. 调用 WinAPI `RegisterHotKey` 注册全局热键
3. 消息循环中监听 `WM_HOTKEY` 消息
4. 收到消息 → 查找对应宏规则 → 传递给播放器模块执行
5. 程序退出时调用 `UnregisterHotKey` 注销所有热键

> ⚠️ **注意：** 监听必须为全局级别，否则程序非前台时无法触发。

---

### 4.3 宏播放器模块（执行模块）

**职责：** 按顺序模拟按键的按下与抬起，保证输入稳定性。

**执行流程：**

```
读取宏序列
  ↓
取出第一个按键单元
  ↓
调用 SendInput → 按下按键
  ↓
等待 KeyDownDuration（按住时长）
  ↓
调用 SendInput → 抬起按键
  ↓
等待 DelayMs（按键间隔）
  ↓
是否还有下一个按键？
  → 是：继续
  → 否：宏结束
```

**时间控制参数（关键）：**

| 参数 | 建议默认值 | 说明 |
|------|-----------|------|
| KeyDownDuration | 20ms | 按键按住时长 |
| DelayMs | 50ms | 两次按键间隔 |

> ⚠️ **输入稳定性是宏软件的核心。** 缺少时间控制会导致漏键、连键、顺序错乱。

**SOP 步骤：**
1. 接收结构化按键序列
2. 遍历序列，依次执行每个按键单元
3. 每个单元：`SendInput(按下)` → 等待 KeyDownDuration → `SendInput(抬起)` → 等待 DelayMs
4. 全部执行完毕，通知监听模块宏已结束

---

### 4.4 配置存储模块

**职责：** 持久化保存热键-宏映射，程序重启后自动恢复。

**存储格式（JSON）：**

```json
{
  "macros": [
    { "hotkey": "F4", "sequence": "DDQQ", "delayMs": 50 },
    { "hotkey": "F5", "sequence": "123",  "delayMs": 50 },
    { "hotkey": "F6", "sequence": "WASD", "delayMs": 30 }
  ],
  "settings": {
    "keyDownDuration": 20,
    "startMinimized": true
  }
}
```

**SOP 步骤：**
1. 程序启动时读取 `config.json`（不存在则创建默认配置）
2. 反序列化为内存中的宏规则列表
3. 用户修改配置后，立即序列化写回 `config.json`
4. 支持热重载（修改后无需重启即可生效）

| 优先级 | 功能 | 说明 |
|--------|------|------|

| P3 | 多宏组 | 类似游戏外设驱动的宏组切换 |

## 八、风险与注意事项

1. **杀毒软件误报**：模拟按键行为（SendInput / WinAPI）可能被部分安全软件标记，建议代码签名。
2. **游戏反作弊**：在受反作弊保护的游戏中使用可能触发封号，需在文档中注明使用限制。
3. **热键冲突**：注册热键前应检测是否已被系统或其他程序占用，注册失败需给出提示。
4. **输入法干扰**：切换输入法状态可能影响字符宏的输出，建议使用虚拟键码（VK_CODE）而非字符直接输入。
