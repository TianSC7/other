# 一、项目目标

开发一个 **Windows 键盘宏播放器软件**，功能如下：

核心功能：

1. 用户定义宏，例如

```
DDQQ
```

2. 设置触发键，例如

```
F4
```

3. 按下 F4 时自动播放

```
D → D → Q → Q
```

4. 软件常驻后台运行。

---

# 二、技术栈

开发环境：

```
.NET 8.0
C#
VS Code
Windows API
```

程序类型建议：

```
Windows 桌面程序
WinForms
```

原因：

* 简单
* 稳定
* 容易托盘运行

# 四、数据结构设计

宏需要一个 **标准数据结构**。

MacroDefinition：

包含：

```
宏名称
触发按键
宏内容
按键延迟
是否循环
```

示例逻辑：

```
Name : "TestMacro"
TriggerKey : F4
Sequence : DDQQ
Delay : 30ms
Loop : false
```

---

# 五、宏解析模块

模块名称：

```
MacroParser
```

职责：

把

```
DDQQ
```

解析成

```
D
D
Q
Q
```

最终形成：

```
按键列表
```

例如：

```
[D, D, Q, Q]
```

解析流程：

```
读取字符串
逐字符解析
转换为键盘键值
返回按键数组
```

---

# 六、宏执行模块

模块：

```
MacroExecutor
```

职责：

按照顺序播放按键。

执行流程：

```
读取按键列表
↓
循环遍历
↓
模拟按下
↓
模拟抬起
↓
等待延迟
↓
继续
```

例如：

```
D
delay
D
delay
Q
delay
Q
```

如果开启循环：

```
播放完成
↓
重新播放
```

---

# 七、键盘模拟模块

模块：

```
KeyboardSimulator
```

职责：

模拟键盘输入。

功能：

```
KeyDown
KeyUp
PressKey
```

执行流程：

```
按下键
↓
短暂停顿
↓
释放键
```

必须有 **延迟控制**，否则可能丢键。

---

# 八、全局热键模块

模块：

```
HotKeyManager
```

职责：

注册系统热键。

流程：

程序启动：

```
读取配置
↓
注册热键
```

监听：

```
用户按F4
↓
触发宏执行
```

注意：

需要支持：

```
多个宏
多个触发键
```

例如：

```
F4 → DDQQ
F6 → AAAA
```

---

# 九、配置管理模块

模块：

```
ConfigManager
```

职责：

读取和保存宏配置。

推荐格式：

```
JSON
```

示例配置：

```
{
  "macros":[
    {
      "name":"Macro1",
      "trigger":"F4",
      "sequence":"DDQQ",
      "delay":30
    }
  ]
}
```

程序启动流程：

```
读取 config.json
↓
加载宏
↓
注册热键
```

---

# 十、主程序流程

启动流程：

```
Program 启动
↓
加载配置
↓
初始化宏引擎
↓
注册热键
↓
进入消息循环
```

运行流程：

```
用户按 F4
↓
HotKeyManager 捕获
↓
MacroEngine 查找宏
↓
MacroExecutor 执行
↓
KeyboardSimulator 输出按键
```
