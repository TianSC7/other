# Phase 4 — 集成 & 测试  分步开发 SOP（.NET 8.0 / WPF）

> 所属总流程：`Quicker_轮盘_开发项目流程.md` → Phase 4
> **前置条件：Phase 1 / 2 / 3 验收清单全部勾选完成**
> 技术栈：.NET 8.0 + xUnit + WPF UI 测试

---

## 阶段目标

Phase 4 不新增业务功能，目标是验证三层打通后**全局无 bug**，并确保性能、边界场景、异常路径均健壮。

```
数据流验收：ConfigService → StateMachine → WheelCanvas
设置联动：  Phase2 保存 → Phase3 热加载 → Phase1 重渲染
边界场景：  多显示器 / 高DPI / 屏幕四角 / 触发键冲突
异常健壮：  配置损坏 / 动作引用失联 / 钩子安装失败
性能：      鼠标延迟 <16ms / 动画 60fps / 内存无泄漏
```

---

## Step 4.1  三层数据流集成验证

### 4.1.1  端到端路径确认

按以下路径逐段确认数据流通畅：

```
[Phase 2 设置界面]
    用户在设置界面绑定某格子动作 → 点击保存
         ↓ ConfigService.Save()
[配置文件 config.json]
         ↓ ConfigService.ConfigSaved 事件
[Phase 3 状态机]
    StateMachine.OnConfigSaved() → ReloadConfig()
         ↓ SceneResolver.BuildSectorDataArray()
[Phase 1 渲染层]
    WheelCanvas.InnerData = ...
    WheelCanvas.SetDisplayOptions(...)
         ↓ InvalidateVisual()
[屏幕上的圆盘]
    下次弹出时显示新绑定的动作
```

**测试步骤**：
1. 启动程序，弹出圆盘，确认内圈北方向显示初始动作
2. 打开设置界面，将内圈北方向改为新动作，保存
3. 再次弹出圆盘，确认内圈北方向已显示新动作
4. 不重启程序，确认热加载生效

**预期结果**：
- 圆盘弹出时立即显示新动作
- 无崩溃、无异常、无空引用
- 配置文件写入成功

### 4.1.2  场景覆盖层验收

验证软件特定场景覆盖全局场景的优先级逻辑：

**测试步骤**：
1. 在设置界面添加一个软件特定场景（如 `chrome.exe`）
2. 为该场景的内圈北方向绑定动作 A
3. 为全局场景的内圈北方向绑定动作 B
4. 切换到 Chrome 窗口，弹出圆盘，确认显示动作 A
5. 切换到其他窗口，弹出圆盘，确认显示动作 B

**预期结果**：
- 软件场景优先级高于全局场景
- 场景切换时圆盘内容正确更新

### 4.1.3  设置项联动验收

验证设置项变更后圆盘行为同步更新：

**测试步骤**：
1. 设置 `ConstrainToScreen=true`，弹出圆盘，确认圆盘不超出屏幕
2. 设置 `ConstrainToScreen=false`，弹出圆盘，确认圆盘可能超出屏幕
3. 设置 `HideLabelWhenIcon=true`，弹出圆盘，确认有图标时不显示文字
4. 设置 `HideLabelWhenIcon=false`，弹出圆盘，确认显示文字

**预期结果**：
- 设置项变更后圆盘行为立即同步
- 无配置丢失或错乱

## Step 4.2  边界场景测试

### 4.2.1  屏幕边缘测试矩阵

| 场景 | 预期行为 |
|------|----------|
| 鼠标在屏幕左上角触发 | 圆盘完全在屏幕内，不超出边界 |
| 鼠标在屏幕右下角触发 | 圆盘完全在屏幕内，不超出边界 |
| 鼠标在任务栏上方触发 | 圆盘不被任务栏遮挡 |
| 鼠标在桌面图标密集区触发 | 圆盘不遮挡图标，可正常交互 |

**测试方法**：
- 在不同位置触发圆盘，观察是否超出屏幕
- 使用 `ConstrainToScreen=true/false` 对比

### 4.2.2  多显示器测试

**测试步骤**：
1. 在主显示器触发圆盘，确认正常显示
2. 在副显示器触发圆盘，确认正常显示
3. 跨显示器移动鼠标，确认圆盘跟随正确

**预期结果**：
- 多显示器环境下圆盘位置计算正确
- 不出现跨显示器错位

### 4.2.3  DPI 缩放测试

**测试步骤**：
1. 设置系统 DPI 为 125%，启动程序，确认圆盘尺寸正确
2. 设置系统 DPI 为 150%，启动程序，确认圆盘尺寸正确
3. 动态切换 DPI，确认圆盘不出现模糊或错位

**预期结果**：
- 圆盘尺寸随 DPI 缩放正确
- 图标和文字清晰不模糊

## Step 4.3  异常路径测试

### 4.3.1  配置文件异常

**测试步骤**：
1. 手动删除 `config.json`，重启程序，确认使用默认配置
2. 手动损坏 `config.json`（如删除大括号），重启程序，确认使用默认配置
3. 手动添加非法动作类型，重启程序，确认忽略非法配置

**预期结果**：
- 配置文件异常时程序不崩溃
- 自动恢复到安全的默认配置

### 4.3.2  动作引用失联

**测试步骤**：
1. 绑定一个不存在的文件路径到 `open` 动作
2. 触发该动作，确认不崩溃，可提示错误
3. 绑定一个不存在的 Quicker 动作 ID
4. 触发该动作，确认不崩溃，可提示错误

**预期结果**：
- 动作执行失败时不崩溃程序
- 可通过日志或提示告知用户

### 4.3.3  钩子安装失败处理

**测试步骤**：
1. 在管理员权限下运行，确认钩子安装成功
2. 在受限环境下模拟钩子安装失败
3. 确认程序有降级处理或提示

**预期结果**：
- 钩子安装失败时程序不崩溃
- 提供用户友好的错误提示

### 4.3.4  并发安全测试

**测试步骤**：
1. 快速连续触发圆盘弹出/关闭
2. 在圆盘显示时快速切换场景
3. 在保存设置的同时触发圆盘

**预期结果**：
- 无竞态条件导致的崩溃
- 状态一致，无数据错乱

## Step 4.4  性能测试

### 4.4.1  鼠标响应延迟

**测试方法**：
- 使用高精度计时器测量从鼠标按下到圆盘显示的时间
- 目标：延迟 < 16ms（1帧）

**预期结果**：
- 圆盘弹出延迟在可接受范围内
- 无明显卡顿

### 4.4.2  动画帧率

**测试方法**：
- 使用性能分析工具监控动画期间的帧率
- 目标：稳定 60fps

**预期结果**：
- 动画流畅，无掉帧
- CPU 占用合理

### 4.4.3  内存泄漏检测

**测试方法**：
- 长时间运行，反复弹出/关闭圆盘
- 使用内存分析工具监控内存使用

**预期结果**：
- 内存使用稳定，无持续增长
- 无资源未释放

## Step 4.5  测试用例清单（TC）

### TC-001  基础触发流程
- **步骤**：启动程序 → 触发圆盘 → 选择动作 → 执行动作
- **预期**：圆盘正常显示，动作正常执行

### TC-002  场景切换
- **步骤**：添加软件场景 → 切换窗口 → 触发圆盘
- **预期**：圆盘内容根据窗口正确切换

### TC-003  超时自动取消
- **步骤**：触发圆盘 → 不操作 → 等待超时
- **预期**：圆盘自动关闭，不执行动作

### TC-004  F1 重复触发
- **步骤**：触发圆盘 → 按 F1 → 再次触发
- **预期**：圆盘重新弹出，位置正确

### TC-005  动作引用失联
- **步骤**：绑定不存在的动作 → 触发
- **预期**：不崩溃，提示错误

### TC-006  配置热加载
- **步骤**：修改设置 → 保存 → 触发圆盘
- **预期**：圆盘立即反映新设置

### TC-007  高 DPI 环境
- **步骤**：在高 DPI 下启动 → 触发圆盘
- **预期**：圆盘尺寸正确，无模糊

### TC-008  边缘触发（ConstrainToScreen=true）
- **步骤**：在屏幕边缘触发
- **预期**：圆盘不超出屏幕
