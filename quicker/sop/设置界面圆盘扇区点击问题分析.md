# 设置界面圆盘扇区点击问题分析

## 问题描述

设置界面的圆盘扇区点击不准确，需要分析渲染和点击检测的匹配问题。

## 两个轮盘的区别

1. **设置轮盘** (SettingsWindow.xaml)
   - 使用 `WheelCanvas` 控件
   - 需要与渲染匹配的无旋转点击检测

2. **主轮盘** (WheelWindow.xaml)
   - 使用相同的 `WheelCanvas` 控件
   - 需要与渲染匹配的有旋转点击检测

## 当前渲染逻辑 (WheelCanvas.cs)

### 扇区中心角度计算

| 模式 | 扇区数量 | 每扇区角度 | 扇区0中心角度 |
|------|---------|-----------|--------------|
| 内圈(环1) | 8 | 45° | 270° + 22.5° = **292.5°** |
| 外圈8模式 | 8 | 45° | 270° + 22.5° = **292.5°** |
| 外圈16模式 | 16 | 22.5° | 270° + 11.25° = **281.25°** |

### 代码位置

- 内圈标签：`WheelCanvas.cs:285` - `centerAngle = (i * innerStep + 270.0 + rotation) % 360.0`
- 外圈16模式标签：`WheelCanvas.cs:323` - `centerAngle = (i * outerStep + 270.0 + 11.25) % 360.0`
- 外圈8模式标签：`WheelCanvas.cs:328` - `centerAngle = (i * outerStep + 270.0 + rotation) % 360.0`

### 旋转常量

```csharp
// WheelConstants.cs
public const double SectorRotationAngle = 22.5;
```

## 当前点击检测逻辑 (SettingsWindow.xaml.cs)

### 当前实现

```csharp
// 内圈 - 8扇区
sector = (int)((angle - 270 + 360) / sectorAngle) % 8;

// 外圈 - 16或8扇区（统一公式）
sector = (int)((angle - 270 + 360) / sectorAngle) % sectors;
```

### 问题

当前点击检测使用统一的 `(angle - 270 + 360)` 公式，假设扇区0中心在270°。

但渲染时：
- 16模式扇区0中心在 **281.25°** (270° + 11.25°)
- 8模式扇区0中心在 **292.5°** (270° + 22.5°)

这导致点击检测的扇区与渲染的扇区**错位**！

## 修复方案

### 方案：还原11.25度偏移

根据用户反馈，设置界面需要还原16模式的11.25度偏移，使点击检测与渲染匹配：

```csharp
if (is16Mode)
{
    // 16模式：扇区0中心在281.25° (270° + 11.25°)
    sector = (int)((angle - 270 + 360 + 11.25) / sectorAngle) % sectors;
}
else
{
    // 8模式：扇区0中心在292.5° (270° + 22.5°)
    sector = (int)((angle - 270 + 360 + 22.5) / sectorAngle) % sectors;
}
```

或者更简单地使用统一公式 + 旋转角度：

```csharp
double rotation = is16Mode ? 11.25 : 22.5; // 扇区偏移
sector = (int)((angle - 270 + 360 + rotation) / sectorAngle) % sectors;
```
