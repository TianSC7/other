# 设置页面问题分析与解决方案

## 问题1：动作页面是上下滑动的，而不是左右

### 当前实现
- 使用 `◀` / `▶` 按钮实现左右导航
- 每个动作页是4x4=16个动作的网格

### 问题原因
- 用户习惯是垂直滚动切换动作页面，而不是点击左右按钮

### 解决方案
```xaml
<!-- 将左右按钮改为上下按钮 -->
<Button Content="▲" Click="OnPrevActionPage"/>  <!-- 向上翻页 -->
<Button Content="▼" Click="OnNextActionPage"/>  <!-- 向下翻页 -->
```

或者使用 ScrollViewer 实现垂直滚动：
```xaml
<ScrollViewer VerticalScrollBarVisibility="Auto">
    <StackPanel>
        <!-- 多个动作页垂直排列 -->
    </StackPanel>
</ScrollViewer>
```

## 问题2：取消按钮上的动作1-1、动作1-2等标题

### 当前实现
在 `OnAddActionPage` 方法中：
```csharp
for (int i = 0; i < ActionsPerPage; i++)
{
    newPage.Add(new SimpleActionItem { Label = $"{pageTitle}-{i + 1}" });
}
```
这会生成"动作1-1"、"动作1-2"等标签。

### 问题
用户希望动作格子上不显示任何预定义标题，保持空白或只显示占位符。

### 解决方案
修改标签生成逻辑：
```csharp
for (int i = 0; i < ActionsPerPage; i++)
{
    newPage.Add(new SimpleActionItem { Label = "" });  // 空白标签
}
```

或者显示占位符 "+"：
```csharp
newPage.Add(new SimpleActionItem { Label = "+" });
```

## 问题3：圆盘点击逻辑不正确

### 当前实现分析

#### 1. 角度计算（SettingsWindow.xaml.cs 第572-577行）
```csharp
double dx = position.X - center.X;
double dy = center.Y - position.Y; // Y轴反转，向上为正
double distance = Math.Sqrt(dx * dx + dy * dy);
double angle = Math.Atan2(dy, dx) * 180 / Math.PI;
if (angle < 0) angle += 360;
```

**问题**：这里的计算是正确的，但需要注意坐标系的对应关系。

#### 2. 缩放比例计算（第579-581行）
```csharp
double scale = Math.Min(WheelPreviewCanvas.ActualWidth, WheelPreviewCanvas.ActualHeight) 
    / WheelConstants.WheelDiameter;
double scaledDistance = distance / scale;
```

**问题**：`WheelDiameter` 在 WheelConstants 中是600（_ring3Radius * 2），但画布只有340x340。
- 画布：340x340
- 轮盘半径（可见的最大）：200（OuterRingRadius）
- WheelDiameter = 600

实际应该使用的比例：
```csharp
double scale = 340.0 / (WheelConstants.OuterRingRadius * 2);  // 340/400 = 0.85
// 而不是
double scale = 340.0 / 600;  // 0.5667 - 这是错误的
```

#### 3. 扇区索引计算（内圈第595-596行）
```csharp
// 内圈8扇区，每个45度
sector = (int)((angle + 22.5) / 45) % 8;
```

**问题**：22.5度偏移量（`SectorRotationAngle`）是用于按钮位置的几何旋转，但点击检测应该直接匹配视觉上的扇区分割线。

#### 4. 外圈扇区计算（第600-604行）
```csharp
ring = "outer";
int sectors = _config.TriggerSettings.OuterRing16Mode ? 16 : 8;
double sectorAngle = 360.0 / sectors;
sector = (int)((angle + sectorAngle / 2) / sectorAngle) % sectors;
```

**问题**：8模式时也加了偏移，但在8模式下分割线没有偏移。

### 核心问题总结

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 缩放比例错误 | WheelDiameter=600，但可见轮盘最大半径是200 | 改为 `2 * OuterRingRadius = 400` |
| 内圈扇区计算偏移 | 22.5度偏移不应该加在点击检测上 | 移除偏移，使用 `(angle / 45) % 8` |
| 外圈8模式偏移 | 8模式下没有旋转偏移 | 根据模式决定是否添加偏移 |

### 修复后的点击检测逻辑

```csharp
private void OnWheelCanvasMouseDown(object sender, MouseButtonEventArgs e)
{
    if (WheelPreviewCanvas == null) return;
    
    var position = e.GetPosition(WheelPreviewCanvas);
    var center = WheelPreviewCanvas.GetCenter();
    
    // 计算距离和角度
    double dx = position.X - center.X;
    double dy = center.Y - position.Y; // Y轴反转
    double distance = Math.Sqrt(dx * dx + dy * dy);
    double angle = Math.Atan2(dy, dx) * 180 / Math.PI;
    if (angle < 0) angle += 360;
    
    // 计算缩放比例 - 使用实际可见的轮盘半径
    double scale = Math.Min(WheelPreviewCanvas.ActualWidth, WheelPreviewCanvas.ActualHeight) 
        / (WheelConstants.OuterRingRadius * 2);  // 340/400 = 0.85
    double scaledDistance = distance / scale;
    
    // 确定圈层和扇区
    string ring = "";
    int sector = 0;
    
    if (scaledDistance < WheelConstants.DeadZoneRadius)
    {
        return; // 死区
    }
    else if (scaledDistance < WheelConstants.InnerRingRadius)
    {
        ring = "inner";
        // 内圈8扇区，从正上方(270°)开始
        // 扇区0对应270°-315°，扇区1对应315°-0°，以此类推
        double sectorAngle = 360.0 / 8;
        sector = (int)((angle + 360 - 270) / sectorAngle) % 8;
    }
    else if (scaledDistance < WheelConstants.OuterRingRadius)
    {
        ring = "outer";
        bool is16Mode = _config.TriggerSettings.OuterRing16Mode;
        int sectors = is16Mode ? 16 : 8;
        double sectorAngle = 360.0 / sectors;
        
        if (is16Mode)
        {
            // 16模式：分割线在0°,22.5°,45°...
            // 扇区中心在11.25°,33.75°... 需要+11.25度偏移
            sector = (int)((angle + 360 - 270 + 11.25) / sectorAngle) % sectors;
        }
        else
        {
            // 8模式：分割线在22.5°,67.5°,112.5°... (旋转22.5度)
            // 扇区中心在45°,90°,135°... 不需要额外偏移
            sector = (int)((angle + 360 - 270) / sectorAngle) % sectors;
        }
    }
    else
    {
        return; // 超出范围
    }
    
    _currentRing = ring;
    _currentSector = sector;
    
    // 更新高亮
    WheelPreviewCanvas.SetHighlight(ring, sector);
}
```

### 高亮绘制的问题

在 WheelCanvas.cs 中，高亮扇形的绘制角度也需要调整：

```csharp
// 内圈高亮绘制（第160-161行）
double centerAngle = (_highlightSector * step + 270.0) % 360.0;

// 外圈16模式高亮绘制（第135行）
double centerAngle = (_highlightSector * step + 270.0 + offset) % 360.0;
```

这里的270度偏移是为了让扇区从正上方开始显示。但点击检测需要匹配这个逻辑。
