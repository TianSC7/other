# Quicker 圆盘菜单（轮盘菜单）详细功能分析

> 本文档面向 AI 辅助开发，目的是让 AI 完整理解 Quicker 轮盘菜单的交互逻辑、数据结构和所有细节，以便正确实现类似功能，避免误解导致错误实现。

---

## 1. 概念定义

**轮盘菜单**（也称"圆盘菜单"）是 Quicker 软件的核心功能之一，是一种以鼠标当前位置为圆心、向四周展开的**放射状快捷操作菜单**。

- 用户按住指定鼠标按键并立即移动鼠标 → 弹出圆盘菜单
- 鼠标移动到某个扇形区域 → 高亮对应动作
- 松开鼠标按键 → 执行该区域绑定的动作
- 整个操作**不需要点击**，完全靠"按住+移动+松开"完成

与普通下拉菜单的本质区别：
- 轮盘菜单出现在**鼠标当前位置**，无需移动鼠标去找菜单
- 所有选项**等距分布**在圆盘上，触发距离相近，更适合盲操作
- 操作流程是**单次连续动作**（按住→滑动→松开），不是"点开菜单再点选项"

---

## 2. 圆盘的结构层次

轮盘共分为**三圈**，从内到外依次为：

```
[内圈 Inner Ring]
  └── 8个扇形格子（方向：上/右上/右/右下/下/左下/左/左上）
[外圈 Outer Ring]
  └── 默认8个扇形格子，可开启16个格子模式
[扩展圈 Extended Ring]
  └── 8个方向区域，触发区域为轮盘外整个屏幕
```

### 2.1 内圈

- 视觉上紧贴圆心的第一圈
- 共 **8 个格子**，方向固定为 8 个基本方向
- 用于放置**最高频**的操作
- 鼠标只需小幅移动即可触发

### 2.2 外圈

- 内圈外侧的第二圈
- 默认 **8 个格子**（同 8 方向）
- 可在设置中开启"允许第二圈使用 16 个动作"，此时格子数变为 **16 个**（每个方向细分为两个扇区）
- 外圈默认**不显示文字标签**（盲操作设计），鼠标悬停时在轮盘正中心显示动作名称和图标
- 目的是让用户"记住方向"而不是"看着菜单找"

### 2.3 扩展圈（第三圈）

- **不在圆盘上显示**，触发区域是轮盘圆盘以外的整个屏幕范围
- 共 **8 个方向区域**，对应 8 个基本方向
- 类似鼠标手势：松开鼠标时，根据鼠标距轮盘圆心的方向判断触发哪个动作
- 适合"划一下"就触发的盲操作，不需要精确位置
- 默认**不可见**，可通过特殊方式让其显示所有格子

## 3. 触发方式

### 3.1 标准触发流程

```
步骤 1：在 Quicker 设置 → 轮盘菜单设置 中，指定一个触发鼠标键
步骤 2：在任意位置按下该鼠标键，并【立即】开始移动鼠标
步骤 3：轮盘菜单在鼠标当前位置弹出
步骤 4：将鼠标滑向目标格子方向（保持按住）
步骤 5：松开鼠标键 → 触发该位置绑定的动作
```

> ⚠️ 关键细节："按下后立即移动"是触发条件。如果按下后没有移动，则视为普通按键使用（不弹出轮盘）。

### 3.2 可用触发按键

- 鼠标中键（滚轮键）
- 鼠标侧键 X1（后退键）
- 鼠标侧键 X2（前进键）
- 侧滚轮（部分鼠标，如罗技 MX Master 系列）
- 通过"高级鼠标触发"设置"键盘引导键 + 鼠标键"的组合触发

> ⚠️ 触发轮盘的鼠标键**独占**，不能同时在其他功能中使用。

### 3.3 重复触发（F1 键）

- 触发轮盘后，将鼠标悬停在某个动作格子上，按 **F1 键**可立即触发该动作
- 可**多次按 F1** 实现重复触发同一动作
- 完成后松开鼠标键退出轮盘
- 重复触发键可在设置中自定义（支持绑定到其他键）
- 还可以进一步绑定"按其他键时执行不同的操作"（版本 1.28.16+）

### 3.4 另类触发方式（通过虚拟键）

- 使用 Quicker 虚拟键 V1（2022年6月之后的版本）
- 先按触发键（X1/X2/侧轮等），再按住右键移动 → 触发轮盘
- 按下触发键后需在 **1 秒内**唤出轮盘，否则触发键状态过期

## 4. 动作设置与管理

### 4.1 场景化动作（上下文感知）

轮盘动作支持**场景化配置**，即根据当前正在使用的软件（进程名）自动切换动作内容：

```
全局场景（Global）
  └── 所有软件下均生效的默认动作
特定软件场景（App-specific）
  └── 在该软件激活时，覆盖对应位置的全局动作
通用场景（Common）
  └── 可附加到其他场景后面
```

优先级规则：
- 特定软件的场景设置 **覆盖** 全局场景中相同位置的设置
- 未在软件场景中设置的位置，**回退**到全局场景的设置

### 4.2 动作管理入口（两个位置）

**位置 1**：Quicker 设置窗口 → 轮盘动作管理页面
- 左侧选择"全局"或某个具体软件场景
- 右侧轮盘示意图中点击/右键空白格子 → 设置该位置动作

**位置 2**：场景与动作管理界面
- 在场景列表选中场景后，可管理该场景下的轮盘菜单

### 4.3 颜色状态说明（管理界面格子颜色含义）

| 颜色 | 含义 |
|------|------|
| 白色 | 全局场景中此位置**未设置**动作 |
| 淡黄色（浅） | 全局场景中此位置**已设置**动作（内圈显示用） |
| 淡黄色（略深） | 全局场景中此位置已设置动作（外圈/可见圈） |
| 高亮/覆盖色 | 当前应用场景中**覆盖**了全局设置的位置 |

### 4.4 从面板拖动动作到轮盘

- 可以直接从 Quicker 主面板的动作格子**拖放**到轮盘管理界面中的位置
- 拖放时建议先"钉住面板"、"锁定动作页切换"，操作更方便
- ⚠️ **重要限制**：拖放到轮盘的动作是**引用（链接）**，不是复制
  - 轮盘存储的是动作页上的动作**引用链接**，不是动作本身
  - **绝对不能**在拖放后删除动作页里的原始动作，否则轮盘动作会"失联"无法使用

## 5. 可绑定的操作类型

轮盘每个格子可以绑定以下类型的操作：

| 操作类型 | 说明 |
|----------|------|
| 发送快捷键 | 模拟按下指定键盘组合键 |
| 模拟输入文本/按键序列 | 格式与"模拟按键B"模块相同，受输入法影响 |
| 粘贴内容 | 以原格式输入，不受输入法影响 |
| 打开文件/文件夹/网址/命令 | 支持 Windows 环境变量（%LOCALAPPDATA% 等） |
| 运行 Quicker 动作 | 触发动作页上已有的 Quicker 动作（引用方式） |
| 发送文本内容 | 支持从本地文件读取，格式：`FILE:d:\路径\文件.txt` |
| 日期时间插入 | 使用 `{0:格式字符串}` 替换符，如 `{0:yyyy-MM-dd}` |

> **动作参数传递**：可以通过不同轮盘位置传递不同参数，让同一个 Quicker 动作响应不同的功能分支。

## 6. 显示与外观设置

### 6.1 设置项一览

| 设置项 | 说明 |
|--------|------|
| 触发按键 | 指定哪个鼠标键触发轮盘 |
| 轮盘尺寸 | 轮盘菜单大小（逻辑单位，非像素） |
| 超时取消 | 触发轮盘后超过设定时间未选定动作则自动取消 |
| 允许第二圈16个动作 | 开启后外圈从8格扩展到16格 |
| 动作有图标时隐藏文字标签 | 有图标则不显示文字名称，保持视觉简洁 |
| 显示限定在屏幕范围内 | 在屏幕边缘触发时，自动把圆盘限制在屏幕内 |
| 自动移动鼠标到轮盘中心 | 配合上一项使用，边缘触发时重定位指针到圆心 |

### 6.2 边缘触发行为

- **不限定屏幕**：轮盘中心就在鼠标位置，部分动作可能超出屏幕范围无法触发
- **限定屏幕**：轮盘圆心自动向屏幕中心偏移，所有动作都在屏幕内可触发
- **+自动移指针**：同时将鼠标指针移动到调整后的轮盘圆心，使方向判断与屏幕中央一致

## 7. 交互细节与注意事项

### 7.1 方向关联设计建议

官方建议每个格子的动作与其方向有语义关联，便于记忆：

| 方向 | 推荐动作语义 |
|------|-------------|
| 左 | 后退、历史 |
| 右 | 前进 |
| 上 | 最小化、向上滚动 |
| 下 | 关闭、确认 |
| 右上/右下等 | 次要操作，自由发挥 |

### 7.2 轮盘 vs 主面板的区别

| 对比维度 | 主面板（方格面板） | 轮盘菜单 |
|----------|-------------------|----------|
| 触发方式 | 按键弹出，然后点击 | 按住+滑动+松开，一气呵成 |
| 操作速度 | 需要找到格子再点击 | 肌肉记忆，盲操作 |
| 方向记忆性 | 弱（格子排列，不直观） | 强（方向与动作语义绑定） |
| 容纳动作数 | 多（多个面板页） | 少（最多内圈8+外圈16+扩展圈8） |
| 适用场景 | 不常用但需要找到的操作 | 最最常用、需要极速触发的操作 |

### 7.3 管理员权限限制

- 轮盘**不支持**直接以管理员身份运行动作
- 如需管理员权限，需在动作页面板上创建动作（设置为管理员运行），再将该动作引用到轮盘上

### 7.4 外圈盲操作设计逻辑

- 外圈默认不显示文字，原因：
  1. 视觉上更简洁，小屏幕不会被挤占
  2. 鼠标悬停时圆盘**中心**会显示当前指向动作的名称和图标
  3. 设计哲学：外圈动作应通过**多次使用形成肌肉记忆**，而不是每次查看
- 强制显示外圈方法：通过"非正常滑动"方式（如组合动作、手势、鼠标高级触发）

## 8. 数据逻辑（AI 实现参考）

### 8.1 数据结构概念模型

```json
{
  "wheel_menu": {
    "settings": {
      "trigger_key": "middle_button | x1 | x2",
      "size": 120,
      "timeout_ms": 0,
      "outer_ring_16_mode": false,
      "hide_label_when_icon": true,
      "constrain_to_screen": true,
      "auto_move_cursor_to_center": false
    },
    "scenes": {
      "global": {
        "inner_ring": {
          "up": { "type": "hotkey", "value": "Ctrl+C" },
          "right": { "type": "run_action", "action_id": "xxx" },
          "down": { "type": "send_text", "value": "你好" },
          "left": null,
          "up_right": null,
          "down_right": null,
          "down_left": null,
          "up_left": null
        },
        "outer_ring": {
          "up": { "type": "open_url", "value": "https://example.com" },
          ...
        },
        "extended_ring": {
          "up": null,
          "right": { "type": "hotkey", "value": "Alt+F4" },
          ...
        }
      },
      "notepad": {
        "inner_ring": {
          "right": { "type": "hotkey", "value": "Ctrl+S" }
        }
      }
    }
  }
}
```

### 8.2 动作解析优先级

```
触发时，对于每个格子位置：
IF 当前软件场景中该位置有设置:
    使用 当前软件场景 的设置
ELSE:
    使用 全局场景 的设置
```

### 8.3 触发流程状态机

```
[IDLE 空闲状态]
    ↓ 按下触发键 + 立即移动鼠标
[WHEEL_SHOWN 轮盘显示]
    ↓ 鼠标移入某扇区
[SECTOR_HIGHLIGHTED 扇区高亮]
    ↓ 松开触发键
[ACTION_TRIGGERED 动作执行]
    → 回到 IDLE

如果在 WHEEL_SHOWN 状态下超时无操作:
    → 自动回到 IDLE（根据"超时取消"设置）

如果在 SECTOR_HIGHLIGHTED 状态下按 F1:
    → ACTION_TRIGGERED（重复触发），不退出轮盘
    → 仍在 WHEEL_SHOWN 状态等待下一步
```

## 9. 常见误区（避免做错）

| 误区 | 正确理解 |
|------|----------|
| 以为轮盘是"点击弹出，再点选项" | 实际是"按住+滑动+松开"一次性完成 |
| 以为外圈动作需要看得到才能用 | 外圈是盲操作设计，靠方向记忆触发 |
| 直接删除动作页上的原始动作 | 轮盘是引用，删原始动作会导致轮盘失联 |
| 认为三圈都始终可见 | 扩展圈默认不可见，触发区为圆盘外整个屏幕 |
| 以为轮盘动作独立存储 | 轮盘存的是对动作页动作的引用链接 |
| 以为可以在轮盘直接管理员运行 | 需要面板动作中转才能以管理员身份运行 |
