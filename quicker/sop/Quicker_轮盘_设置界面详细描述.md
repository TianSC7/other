# Quicker 轮盘菜单 — 设置界面详细描述

> 配套文档：`Quicker_圆盘菜单功能分析.md` · `Quicker_轮盘菜单_界面尺寸详细描述.md`
> 本文档专门描述**设置界面（Phase 2）**的所有 UI 元素、布局、交互行为和数据结构，供 AI 精确实现设置界面，避免遗漏或做错。

---

## 一、设置界面整体结构

设置界面分为**两个独立入口**，功能相同，从不同位置打开：

```
入口 A：Quicker 主设置窗口 → 左侧导航 → "轮盘菜单" 分组
         └── 轮盘菜单设置页（全局参数）
         └── 轮盘菜单动作管理页（格子绑定）

入口 B：场景与动作管理窗口
         └── 选中某个场景 → 右侧显示该场景的轮盘格子配置
```

两个入口操作的是**同一份配置数据**，保存后互相可见。

### 3.2 各控件详细说明

#### 触发鼠标键（下拉选择框）

- 控件类型：ComboBox / Select
- 宽度：约 220 px
- 选项列表：

| 选项值 | 显示文字 |
|--------|---------|
| `middle` | 鼠标中键（滚轮键）|
| `x1` | 鼠标侧键 X1（后退键）|
| `x2` | 鼠标侧键 X2（前进键）|
| `side_scroll` | 侧滚轮（罗技 MX Master 等）|
| `advanced` | 高级鼠标触发（跳转到高级设置页）|

- 默认值：`middle`
- 选择 `advanced` 时：当前下拉框禁用，页面底部出现跳转链接/按钮
- 冲突检测：若所选按键已被其他功能占用，在下拉框下方显示红色警告文字

#### 轮盘尺寸（滑块 + 数字输入）

- 控件：水平滑块 + 右侧数字输入框
- 滑块范围：50 ~ 300（逻辑单位）
- 数字输入范围：50 ~ 300，整数
- 默认值：120
- 两个控件双向绑定，改一个另一个同步
- 实时预览：修改后轮盘示意图（动作管理页中）同步缩放（可选实现）

#### 超时取消（数字输入）

- 控件：数字输入框，宽约 80 px
- 单位：毫秒（ms）
- 范围：0 ~ 30000
- 默认值：0
- 0 的特殊含义：**不超时**，输入框旁边显示灰色说明文字"0 = 不超时"
- 不接受负数

#### 功能开关（Toggle / Checkbox）

| 开关 | 默认状态 | 说明 |
|------|---------|------|
| 允许第二圈使用16个动作 | 关 | 开启后动作管理页外圈格子数变为16 |
| 动作有图标时隐藏文字标签 | 开 | 影响圆盘渲染层 |
| 显示位置限定在屏幕范围内 | 开 | 边缘触发时圆盘自动偏移 |
| 自动移动鼠标到轮盘中心 | 关 | **灰色禁用状态**，当上一项关闭时 |

- 开关样式：系统 Toggle 或 Checkbox，高度约 22 px
- 行间距：约 8 px
- 灰色禁用逻辑：`自动移动鼠标` 与 `限定屏幕范围` 联动，后者关闭时前者 `disabled`

#### 重复触发键（按键录入框）

- 控件：只读文本框（点击后进入录制模式）+ 右侧清除按钮
- 文本框宽约 120 px，显示当前绑定键名（如 "F1"、"Ctrl+Q"）
- 点击文本框 → 进入录制模式，边框变蓝，提示文字"请按下目标键..."
- 录制模式下按任意键 → 记录并退出录制模式
- 清除按钮：`×`，宽约 28 px，点击后清空绑定（禁用重复触发功能）
- 默认值：`F1`

## 四、轮盘动作管理页

### 4.1 页面整体布局

```
┌─────────────────────────────────────────────────────────────────┐
│  轮盘菜单动作管理                                                 │
├──────────────┬──────────────────────────────────────────────────┤
│  场景列表    │  轮盘可视化编辑区                                   │
│  [约220px]   │  [剩余宽度]                                        │
│              │                                                   │
│  🌐 全局      │   ┌──── 圆盘示意图（中央） ────┐                  │
│  📄 记事本    │   │                            │                  │
│  📄 Chrome   │   │    [可视化圆盘，可点击]      │                  │
│  📄 VSCode   │   │                            │                  │
│  ...         │   └────────────────────────────┘                  │
│              │                                                   │
│  [+ 添加场景] │   [圈层切换Tab：内圈 | 外圈 | 扩展圈]             │

└──────────────┴──────────────────────────────────────────────────┘
```

### 4.2 场景列表（左侧栏）

- 宽度：约 220 px，固定，不可拖拽
- 每行高度：约 32 px
- 第一行固定为 **全局场景**，图标：🌐，不可删除
- 其余行为软件特定场景，图标：该软件的 EXE 图标（提取系统图标）或默认文件图标
- 当前选中行：高亮背景色
- 右键场景行 → 上下文菜单：
  - 重命名（修改显示名称，进程名不变）
  - 删除（有确认弹窗）
  - 复制场景配置到…
- 底部固定按钮：`+ 添加场景`，点击弹出"添加场景"对话框

#### 添加场景对话框

```
┌────────────────────────────────────┐
│  添加软件场景                        │
│                                    │
│  进程名：[输入框           ]        │
│         如：notepad.exe            │
│                                    │
│  或从当前运行进程选择：              │
│  ┌────────────────────────────┐   │
│  │ notepad.exe - 记事本        │   │
│  │ chrome.exe - Google Chrome │   │
│  │ ...                        │   │
│  └────────────────────────────┘   │
│                                    │
│              [确定]  [取消]         │
└────────────────────────────────────┘
```

- 进程名不区分大小写，存储时统一转换为小写
- 列表实时刷新当前运行进程
- 点击列表行自动填充进程名输入框

### 4.3 圆盘可视化编辑区（中央核心区域）

#### 圆盘示意图尺寸

- 示意图直径：约 340 px（比实际轮盘略大，便于点击操作）
- 圆心：编辑区中央
- 三圈比例与实际轮盘完全一致（20:80:160 的比例关系）

#### 圆盘示意图格子颜色规则

| 格子状态 | 颜色 |
|---------|------|
| 全局场景 - 未设置动作 | 白色（`#ffffff`）|
| 全局场景 - 已设置动作（内圈） | 淡黄色浅（`#fffde7`）|
| 全局场景 - 已设置动作（外圈） | 淡黄色深（`#fff9c4`）|
| 软件场景 - 覆盖了全局此位置 | 浅蓝色（`#e3f2fd`）|
| 软件场景 - 未覆盖（显示全局兜底）| 白色（显示全局动作名，但字体灰色表示"继承"）|
| 任意场景 - 鼠标悬停 | 边框加深，背景略深一色阶 |
| 任意场景 - 当前选中 | 蓝色边框（`2px solid #1976d2`）|

#### 圆盘示意图格子内内容

- 有动作的格子：显示动作名称（约 10 px 字体，单行，超长省略）
- 有图标的动作：图标（约 16×16 px）+ 名称（图标下方）
- 无动作的格子：显示 `+`（灰色，提示可添加）

#### 圆盘格子交互

- **鼠标悬停**：边框高亮，底部状态栏显示该格子位置信息（如"内圈 · 右上方向"）
- **左键单击**：选中格子，右侧显示格子详情编辑区
- **左键双击**：直接打开动作绑定编辑弹窗
- **右键单击**：弹出上下文菜单（见下方）

#### 右键上下文菜单

```
┌────────────────────────┐
│  设置动作...            │  ← 打开编辑弹窗
│  清除动作              │  ← 清空该格子
│  ────────────────────  │
│  复制                  │
│  粘贴                  │  ← 若剪贴板中有动作配置
│  ────────────────────  │
│  测试执行              │  ← 立即执行该格子动作（调试用）
└────────────────────────┘
```

### 4.4 圈层切换 Tab

位置：圆盘示意图正下方或正上方
样式：3 个 Tab 标签

```
[内圈 (8格)]  [外圈 (8/16格)]  [扩展圈 (8格)]
```

- 切换 Tab 时，圆盘示意图切换显示对应圈层的格子
- 扩展圈 Tab：由于扩展圈不可见，示意图以虚线方框 + 方向箭头表示8个方向区域
- 外圈 Tab：若开启了16格模式，Tab 标签变为 `[外圈 (16格)]`

### 4.5 格子详情编辑区（右侧或底部面板）

点击格子后显示：

```
┌───────────────────────────────────────┐
│  内圈 · 右上方向（NE）                  │
├───────────────────────────────────────┤
│  动作类型：[下拉选择框         ▼]       │
│                                       │
│  （根据动作类型动态显示对应输入区域）     │
│                                       │
│  [保存到此格子]  [清除此格子]           │
└───────────────────────────────────────┘
```

#### 动作类型下拉选项

| 选项值 | 显示文字 | 显示的附加输入 |
|--------|---------|--------------|
| `hotkey` | 发送快捷键 | 按键录入框 |
| `simulate_input` | 模拟输入文本/按键序列 | 多行文本框 |
| `paste` | 粘贴内容 | 多行文本框 |
| `open` | 打开文件/文件夹/网址/命令 | 单行文本框 + 浏览按钮 |
| `run_action` | 运行 Quicker 动作 | 动作选择器（见下方）|
| `send_text` | 发送文本内容 | 多行文本框 + FILE: 说明 |
| `datetime` | 插入日期时间 | 格式字符串输入框 + 预览 |
| `none` | （不设置）| 无 |

#### 各动作类型的输入区域详细

**发送快捷键**
```
  快捷键：[按键录入框  Ctrl+C  ] [清除]
```

**模拟输入文本/按键序列**
```
  内容：┌────────────────────────────┐
        │ {ctrl}{c}                  │  ← 多行文本框，支持特殊键名
        └────────────────────────────┘
  提示：使用 {键名} 格式表示按键，如 {enter} {tab} {ctrl}
```

**粘贴内容**
```
  内容：┌────────────────────────────┐
        │ 要粘贴的文字内容            │
        └────────────────────────────┘
  说明：以原格式输入，不受输入法影响
```

**打开文件/文件夹/网址/命令**
```
  路径：[输入框                         ] [浏览...]
  说明：支持环境变量，如 %LOCALAPPDATA%\xxx
```

**运行 Quicker 动作**
```
  选择动作：[动作选择器下拉/弹窗        ]
            显示：动作页名称 > 动作名称
  传参：   [参数输入框（可选）          ]
  注意：⚠️ 删除原始动作后此处会失联
```

**发送文本内容**
```
  内容：┌────────────────────────────┐
        │ 要发送的文字                │
        └────────────────────────────┘
  或从文件读取：[FILE:] [路径输入框    ] [浏览]
  日期替换符：{0:yyyy-MM-dd HH:mm:ss}
```

**插入日期时间**
```
  格式字符串：[{0:yyyy-MM-dd}         ]
  预览：      2024-01-15              ← 实时显示当前时间格式化结果
  常用格式：  [yyyy-MM-dd] [HH:mm:ss] [yyyy年MM月dd日]（快速填入按钮）
```

### 数据读写规则

- 文件位置：`%APPDATA%\Quicker\wheel_config.json`（参考路径）
- 读取时机：设置界面打开时读取一次；功能逻辑层在每次触发轮盘前也读取（或监听文件变化）
- 写入时机：用户点击"保存"或"确定"后写入；**不自动保存**
- 软件场景中未出现的格子方向 = 继承全局场景（不需要存储 null，缺省即继承）
- 外圈16格模式开启时，方向键名扩展为 `N`, `NNE`, `NE`, `ENE`, `E`, `ESE`, `SE`, `SSE` 等16方向

## 七、设置界面交互细节

### 7.1 未保存变更提示

- 设置页有修改但未保存时，左侧导航该页标题显示 `*`（如 `轮盘菜单设置 *`）
- 切换到其他页面时弹窗提示："有未保存的修改，是否保存？[保存] [不保存] [取消]"
- 关闭整个设置窗口时同样提示

### 7.2 触发键冲突检测

- 选择触发键后立即检测：遍历所有已配置功能（面板触发键、手势触发键等）
- 有冲突时：下拉框下方显示红色警告 `⚠️ 此按键已被"xxx功能"使用，可能产生冲突`
- 不阻止保存，仅警告

### 7.3 外圈格子数切换

- 从 8格 → 16格：新增的 8 个格子初始化为空（null）
- 从 16格 → 8格：**警告弹窗** "切换将清除外圈中 8 个子格子的动作配置，是否继续？"
  - 确认后：保留原 8 个主方向格子，删除 8 个细分格子
  - 取消后：开关恢复原状态

### 7.4 动作选择器（运行 Quicker 动作类型）

- 弹出一个次级弹窗，列出所有动作页及其中的动作
- 树形结构：动作页（根节点）→ 动作（叶节点）
- 支持搜索过滤
- 选中动作后记录其 **唯一 ID**（而非名称），防止重命名后失联
- 若目标动作已被删除：在编辑区显示红色提示 `⚠️ 目标动作已不存在，请重新绑定`

### 7.5 "从面板拖放" 功能

- 设置界面打开时，可与 Quicker 主面板并排显示
- 从主面板的动作格子**拖放到设置界面圆盘示意图的格子**上
- 拖放成功：格子更新为该动作的引用（显示动作图标 + 名称）
- 操作提示：拖放前建议"钉住面板"防止面板消失
- ⚠️ 重要：拖放是引用不是复制，原始动作删除后轮盘失联

## 九、常见实现错误（避免踩坑）

| 错误做法 | 正确做法 |
|---------|---------|
| 直接存动作名称作为引用 | 存动作唯一 ID，名称仅作显示用 |
| 软件场景用完整路径匹配 | 只用进程名（不含路径）匹配 |
| 16格→8格直接截断不提示 | 必须弹窗警告，用户确认后才执行 |
| 设置修改后立即写文件 | 等用户点保存后再写入 |
| 动作失联时崩溃 | 失联时显示警告提示，不影响其他格子 |
| 外圈默认显示文字标签 | 外圈仅在圆心区显示提示，格子本身不显文字 |
| 日期时间格式不预览 | 必须实时预览当前时间的格式化效果 |
