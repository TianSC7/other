# 动作管理页开发问题分析

## 发现的关键问题

### 项目中有两个SettingsWindow！

1. **旧版本**（用户正在使用的）：`WheelMenu/Windows/SettingsWindow.xaml`
   - 有"内圈·北(N)"等旧元素
   - 使用简单的左右两栏布局
   - 没有页面切换功能

2. **新版本**（我创建的）：`WheelMenu/Settings/Views/SettingsWindow.xaml`
   - 有4×4网格、动作页面标题等新元素
   - 有导航按钮可以切换页面
   - 还未与主程序关联

### 2. ActionManagerPageViewModel.cs 关键代码

**Outer16Mode属性**:
```csharp
public bool Outer16Mode
{
    get => _config.Settings.OuterRing16Mode;
    set
    {
        if (_config.Settings.OuterRing16Mode == value) return;
        _config.Settings.OuterRing16Mode = value;
        OnPropertyChanged();
        RebuildRing2Sectors();
        MarkDirty();
    }
}
```

## 需要解决的问题

旧的`WheelMenu/Windows/SettingsWindow.xaml`需要完全重写或者集成新的ActionManagerPage。

当前旧界面的结构：
```
- 左侧：内圈/外圈/扩展圈选择 + 8个方向选择（北、东北、东...）
- 右侧：圆盘示意图 + 格子编辑区（动作名称、动作类型等）
```

需要改为新界面：
```
- 左侧：场景列表（全局、通用、任务栏、桌面等）
- 右侧：
  - 动作页面区：标题 + 添加动作页按钮 + 动作页Tab + 4×4动作网格
  - 轮盘圆盘区：外圈16格开关 + 圆盘预览（可点击/拖拽）
```

## 问题已解决 ✅

**解决方案**：已修改 `WheelMenu/App.xaml.cs`

将：
```csharp
private Windows.SettingsWindow? _settingsWindow;
// ...
_settingsWindow = new Windows.SettingsWindow();
```

改为：
```csharp
private Settings.Views.SettingsWindow? _settingsWindow;
// ...
_settingsWindow = new Settings.Views.SettingsWindow();
```

现在程序启动设置界面时，将使用新版本的 SettingsWindow（`WheelMenu/Settings/Views/SettingsWindow.xaml`），该版本包含：
- 4×4动作网格
- 动作页面切换功能
- 外圈16格开关

---

## 结论

代码已经创建完成，但由于项目使用了两个不同的SettingsWindow文件，用户看到的是旧版本。

**解决方案**：需要将新的ActionManagerPage集成到Windows/SettingsWindow中，或者让程序使用Settings/Views下的新SettingsWindow。
