# 高亮扇区偏移问题分析

## 问题描述
当环2切换为16扇区时，分割线和高亮扇区位置不匹配。分割线在高亮扇区中间。

## 问题分析

### 当前16模式分割线角度
```
0°, 22.5°, 45°, 67.5°, 90°, 112.5°, 135°, 157.5°, 180°, 202.5°, 225°, 247.5°, 270°, 292.5°, 315°, 337.5°
```

### 当前高亮扇区计算（需要检查代码）
高亮扇区中心角度 = `sectorIndex * step + 270°`

其中 step = 360° / 16 = 22.5°

所以扇区中心角度：
- sectorIndex=0: 0 * 22.5 + 270 = 270°
- sectorIndex=1: 1 * 22.5 + 270 = 292.5°
- sectorIndex=2: 2 * 22.5 + 270 = 315°
- ...

### 问题原因
分割线在 0°, 22.5°, 45°...
但高亮扇区中心在 270°, 292.5°, 315°...

这两组角度不匹配！分割线应该在扇区之间，而不是在扇区中心。

### 解决方案
高亮扇区中心应该偏移11.25度（分割线角度的一半），这样：
- 分割线在 0°, 22.5°, 45°...
- 扇区中心在 11.25°, 33.75°, 56.25°...

这样分割线正好在扇区之间。

## 需要修改的代码
高亮扇区计算时，16模式需要额外偏移11.25度。
