# 轮盘点击与高亮不匹配问题分析

## 问题描述
轮盘点击扇区和显示高亮的位置不一致，点击某个扇区时，高亮显示在错误的位置。

## 问题根源分析

### 1. 核心几何参数对比

| 参数 | DiskUI 示例 | WheelMenu 当前 |
|------|-------------|---------------|
| 起始角度 START | -90° | 无 (0°) |
| 旋转角度 ROT | 22.5° | 22.5° |
| 分割线角度 | -90 + i×45° | 22.5 + i×45° |

### 2. 代码流程分析

#### 2.1 绘制高亮流程 (WheelCanvas.cs)

```csharp
// DrawInnerRing() - 第146-170行
if (_highlightRing == "inner" && _highlightSector >= 0)
{
    double step = 360.0 / WheelConstants.InnerSectors;
    double rotation = WheelConstants.SectorRotationAngle; // 22.5度
    double centerAngle = (_highlightSector * step + rotation) % 360.0;
    double startAngle = centerAngle - step / 2.0;
    double endAngle = centerAngle + step / 2.0;
    // 绘制扇形...
}
```

**问题1**: 高亮角度计算使用 `rotation = 22.5°`
- 扇区0的中心角度 = 0×45 + 22.5 = 22.5°
- 扇区1的中心角度 = 1×45 + 22.5 = 67.5°

#### 2.2 点击测试流程 (WheelGeometry.cs)

```csharp
// HitTestMainWheel() - 第180-226行
double angleDeg = Math.Atan2(dx, -dy) * 180.0 / Math.PI; // 鼠标角度
if (angleDeg < 0) angleDeg += 360.0;

double rotationAngle = WheelConstants.SectorRotationAngle; // 22.5度

// 应用旋转角度
double adjustedAngle = (angleDeg - rotationAngle + 360.0) % 360.0;
double step = 360.0 / totalSectors;
int sector = (int)((adjustedAngle + step / 2.0) / step) % totalSectors;
```

**问题2**: 点击测试中的角度调整
- 鼠标在0°方向（正上方）时
- adjustedAngle = 0 - 22.5 = -22.5 → 337.5°
- sector = (337.5 + 22.5) / 45 = 8 % 8 = 0

**关键发现**: 点击测试和高亮绘制使用的是不同的坐标系！

### 3. 逐行代码分析

#### 3.1 绘制侧 (WheelCanvas.cs)

```csharp
// 第284-290行 - DrawLabels
double centerAngle = (i * innerStep + 270.0 + rotation) % 360.0;
```

- 标签绘制使用 `270 + rotation` = 270 + 22.5 = 292.5°
- 这与高亮角度 22.5° 相差 270°！

```

#### 3.2 点击侧 (WheelGeometry.cs)

// WPF坐标系: 0°=右, 90°=下
// 轮盘坐标系: 0°=上, 顺时针

```

**问题3**: 坐标系转换不一致！

#### 3.3 HitTestMainWheel 详细分析

```csharp
// 第191行 - 获取鼠标角度（WPF坐标系）
double angleDeg = Math.Atan2(dx, -dy) * 180.0 / Math.PI;
// atan2(dx, -dy) 产生: 0°=上, 90°=右, 180°=下, 270°=左 (顺时针)

// 第195-218行 - 根据圈层设置旋转角度
double rotationAngle = WheelConstants.SectorRotationAngle; // 22.5度
// 问题: 外圈使用 33.75度，内圈使用 22.5度，不一致！

// 第221-223行 - 计算扇区索引
double adjustedAngle = (angleDeg - rotationAngle + 360.0) % 360.0;
double step = 360.0 / totalSectors;
int sector = (int)((adjustedAngle + step / 2.0) / step) % totalSectors;
```

### 4. 问题总结

#### 问题A: 坐标系不统一
- 绘制侧: 使用 `270 + rotation` 计算标签中心
- 高亮侧: 使用 `rotation` 计算扇区中心
- 点击侧: 使用 `atan2(dx, -dy)` + rotation

#### 问题B: 旋转角度不统一
- 内圈: rotation = 22.5°
- 外圈(旧): rotation = 33.75° (22.5 + 11.25)
- 外圈(修改后): rotation = -67.5° (与内圈对齐)

#### 问题C: 角度调整公式不一致
- 绘制高亮: `centerAngle = sector * step + rotation`
- 点击测试: `adjustedAngle = mouseAngle - rotation`

### 5. 修复方案

#### 方案: 统一使用 -67.5° 起始角度（推荐）

参考 DiskUI 示例:
- 起始角度: START = -90°
- 旋转角度: ROT = 22.5°
- 第一分割线角度: START + ROT = -67.5°
- 扇区角度: -67.5 + i×45° + 扇区半宽22.5°

**修复后的代码**:

```csharp
// WheelConstants.cs
public const double SectorStartAngle = -90.0;
public const double SectorRotationAngle = 22.5;

// WheelCanvas.cs - 所有绘制方法使用
double startAngle = WheelConstants.SectorStartAngle + WheelConstants.SectorRotationAngle; // -67.5度

// WheelGeometry.cs - 点击测试使用相同的角度
double rotationAngle = WheelConstants.SectorStartAngle + WheelConstants.SectorRotationAngle;
```

### 7. 发现的严重问题：点击测试逻辑不统一！

#### 问题A: atan2 参数顺序不一致！

**SettingsWindow.xaml.cs 第643行**:
```csharp
double angle = Math.Atan2(dy, dx) * 180 / Math.PI;
```

**WheelGeometry.cs 第106行**:
```csharp
double angleDeg = Math.Atan2(dx, -dy) * 180.0 / Math.PI;
```

- `Atan2(dy, dx)`: 标准数学坐标系，0°在正右方
- `Atan2(dx, -dy)`: 轮盘坐标系，0°在正上方（因为Y轴向下）

这是两个完全不同的坐标系！

#### 问题B: 扇区计算公式不一致！

**SettingsWindow.xaml.cs 第665行 (内圈)**:
```csharp
sector = (int)((angle - 270 + 360) / sectorAngle) % 8;
```

**WheelGeometry.cs (内圈)**:
```csharp
double adjustedAngle = (angleDeg - rotationAngle + 360.0) % 360.0;
int sector = (int)((adjustedAngle + step / 2.0) / step) % totalSectors;
```

公式完全不同！

#### 问题C: 旋转角度使用不一致！

**SettingsWindow.xaml.cs 第676-677行 (外圈)**:
```csharp
double rotation = is16Mode ? 11.25 : 22.5;
sector = (int)((angle - 270 + 360 + rotation) / sectorAngle) % sectors;
```

**WheelGeometry.cs (外圈)**:
```csharp
rotationAngle = WheelConstants.SectorStartAngle + WheelConstants.SectorRotationAngle; // -67.5度
```

### 8. 根本原因总结

项目中存在**三套不同的点击测试逻辑**：

1. **WheelGeometry.HitTestMainWheel()** - 用于主轮盘
2. **SettingsWindow.OnWheelCanvasMouseDown()** - 用于设置界面预览
3. **WheelCanvas 内部的绘制逻辑** - 用于高亮显示

这三套逻辑使用不同的角度计算公式，导致点击和高亮不匹配！

### 9. 修复方案

需要统一所有点击测试逻辑，推荐使用 WheelGeometry.cs 中的逻辑，因为我已经修复它使用正确的起始角度。

**SettingsWindow.xaml.cs 需要修改 OnWheelCanvasMouseDown 方法**，改用 WheelGeometry 的方法：

```csharp
private void OnWheelCanvasMouseDown(object sender, MouseButtonEventArgs e)
{
    if (WheelPreviewCanvas == null) return;
    
    var position = e.GetPosition(WheelPreviewCanvas);
    var center = WheelPreviewCanvas.GetCenter();
    
    // 使用 WheelGeometry 的统一逻辑
    bool is16Mode = _config?.TriggerSettings?.OuterRing16Mode ?? false;
    int sectors = is16Mode ? 16 : 8;
    
    var (ring, sector) = WheelGeometry.HitTestMainWheel(
        center.X, center.Y,
        position.X, position.Y,
        sectors,
        WheelPreviewCanvas.GetExtendedRingVisible());
    
    // 处理点击...
}
```

它使用的是独立的扇区计算逻辑（第632-680行），而不是调用 WheelGeometry 的统一方法。
