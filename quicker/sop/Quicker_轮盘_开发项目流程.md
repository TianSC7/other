# Quicker 轮盘菜单 — 开发项目流程 SOP

> 开发顺序：**圆盘界面 → 设置界面 → 功能逻辑 → 集成测试**
> 三个主阶段相互独立，可单独验收，最后由第四阶段统一集成。

---

## 总体原则

每个 Phase 完成后应可**独立运行并肉眼验证**，不依赖后续阶段。这样 AI 可以分段生成、分段检查，不会因为逻辑未接入就无法看到效果。

```
Phase 1（圆盘渲染）→ 能看到圆盘，但动作是假数据
Phase 2（设置界面）→ 能配置参数，写入本地，但不驱动圆盘
Phase 3（功能逻辑）→ 接入真实触发与动作，驱动 Phase1 渲染
Phase 4（集成测试）→ 三层打通，全功能验收
```

---

## Phase 1  圆盘界面层（纯视觉渲染）

**目标**：只管画，不管交互逻辑，用假数据填充所有格子。

### Step 1.1  搭建圆盘基础几何结构

- 创建无边框、透明背景窗口（覆盖触发区域）
- 确定圆心坐标（初始阶段可硬编码为屏幕中心）
- 建立三圈半径常量：
  - 圆心死区半径：20 px
  - 内圈外半径：80 px
  - 外圈外半径：160 px
- 选择渲染方案（Canvas 2D / SVG / 系统 GDI+），确保可绘制透明扇形

**验收标准**：能在屏幕上显示出正确比例的三层同心圆区域。

---

### Step 1.2  扇形格子绘制 + 分隔线

- 内圈绘制 8 个等分扇形（45° 每格）
- 外圈绘制 8 个等分扇形（默认模式）；预留 16 格模式切换接口
- 绘制放射状分隔线，线宽 1 px，颜色 `rgba(0,0,0,0.12)`
- 实现格子三种颜色状态：
  - 空格：透明
  - 有动作（未选中）：半透明浅色填充
  - 高亮（鼠标悬停）：高亮填充色
- 实现半透明圆形整体背景遮罩（`rgba(255,255,255,0.88)`）

**验收标准**：格子轮廓清晰，8 方向分布均匀，高亮色可用假事件手动切换测试。

---

### Step 1.3  格子内元素渲染

- 内圈格子内渲染动作图标（24×24 px，居中对齐）
- 图标下方渲染动作名称文字（11 px，单行，超长省略号）
- 圆心区域（半径 20 px 的圆）：悬停外圈时显示图标（20×20 px）+ 名称
- 实现三种显示模式：
  - 有图标 + 有文字（默认）
  - 有图标 + 隐藏文字（设置项控制）
  - 无图标 + 只显文字（略大字体）
- 用假图标和假名称填充所有格子验证布局

**验收标准**：图标和文字在扇形内布局正确，圆心提示区随悬停格子更新。

---

### Step 1.4  弹出 / 关闭动画 + 屏幕边缘处理

- 弹出动画：`scale(0.3 → 1.0) + opacity(0 → 1)`，时长 120 ms，ease-out
- 关闭动画：`scale(1.0 → 0.5) + opacity(1 → 0)`，时长 80 ms，ease-in
- 屏幕边缘偏移逻辑：
  - 模式 A：不处理（圆心在鼠标位置）
  - 模式 B：检测边缘，自动将圆心向内偏移，确保整个 320 px 圆盘在屏幕内
  - 模式 C：模式 B + 同步将鼠标指针移动到新圆心位置

**验收标准**：动画流畅，在屏幕四角触发时圆盘不超出边界。

---

## Phase 2  设置界面层（配置 UI）

**目标**：提供完整的设置操作界面，读写本地配置文件，与 Phase 1 圆盘渲染暂不联动。

### Step 2.1  轮盘动作管理面板

- 渲染可视化圆盘示意图（与实际轮盘同比例，但为静态 UI 元素）
- 实现格子颜色状态区分：
  - 白色 → 全局场景未设置
  - 淡黄（浅） → 全局已设置（内圈）
  - 淡黄（深） → 全局已设置（外圈）
  - 高亮覆盖色 → 当前软件场景覆盖了全局
- 点击格子 → 弹出动作绑定编辑框（支持所有动作类型）
- 右键格子 → 上下文菜单（清除、复制、粘贴）
- 支持从外部拖放动作到格子（绑定为引用，非复制）

**验收标准**：格子颜色状态正确，编辑后写入本地配置文件可持久化。

---

### Step 2.2  全局参数设置面板

提供以下配置项 UI 控件：

| 设置项 | 控件类型 |
|--------|---------|
| 触发鼠标键 | 下拉选择（中键/X1/X2/侧轮/组合键）|
| 轮盘尺寸 | 数字输入 + 滑块 |
| 超时取消时间 | 数字输入（0 = 不超时）|
| 外圈 16 格模式 | 开关 Toggle |
| 有图标时隐藏文字 | 开关 Toggle |
| 显示限定在屏幕内 | 开关 Toggle |
| 自动移鼠标到圆心 | 开关 Toggle（依赖上一项）|

**验收标准**：所有设置项修改后写入配置，重启后可恢复。

---

### Step 2.3  场景管理面板

- 左侧列表：全局场景 + 已添加的软件场景（按进程名）
- 添加软件场景：输入进程名（如 `notepad.exe`）或从当前运行进程选择
- 选中场景后右侧圆盘示意图切换显示该场景的动作配置
- 实现优先级逻辑：软件场景中设置的格子显示覆盖色，未设置格子显示全局兜底内容
- 删除场景时确认提示

**验收标准**：切换场景后格子内容正确显示，保存后配置文件正确分层存储。

---

## Phase 3  功能逻辑层（交互与动作执行）

**目标**：实现真实的鼠标监听、方向判断、场景切换和动作执行，接入 Phase 1 渲染层。

### Step 3.1  鼠标钩子 + 触发状态机

实现以下状态机：

```
IDLE
  ↓ [触发键按下 + 鼠标开始移动]
WHEEL_SHOWN          ← 此时调用 Phase1 弹出动画
  ↓ [鼠标进入某扇区]
SECTOR_HIGHLIGHTED   ← 此时更新 Phase1 高亮状态
  ↓ [松开触发键]
ACTION_TRIGGERED     ← 执行绑定动作
  → 回到 IDLE        ← 此时调用 Phase1 关闭动画

异常路径：
WHEEL_SHOWN + 超时 → IDLE（调用关闭动画）
SECTOR_HIGHLIGHTED + 按 F1 → ACTION_TRIGGERED（不退出，留在 WHEEL_SHOWN）
```

- 安装全局鼠标钩子（低级鼠标钩子）
- 区分"触发键按下后有移动"与"触发键单击无移动"（透传给系统）

**验收标准**：按住中键移动可呼出圆盘，松开能正确触发对应位置，单击不呼出。

---

### Step 3.2  方向判断算法

```python
dx = mouse.x - center.x
dy = mouse.y - center.y       # Y 轴向下为正

angle = atan2(dx, -dy)        # 以正上方为 0°，顺时针
if angle < 0: angle += 360

# 圈层判断
dist = sqrt(dx² + dy²)
if dist <= 20:   zone = "center_dead"
elif dist <= 80: zone = "inner"
elif dist <= 160: zone = "outer"
else:            zone = "extended"

# 扇区判断（8格）
sector = floor((angle + 22.5) / 45) % 8
# 扇区判断（16格外圈）
sector = floor((angle + 11.25) / 22.5) % 16
```

- 实时计算并通知 Phase1 更新高亮格子
- 圆心死区内不高亮任何格子

**验收标准**：鼠标移向 8 个方向时对应格子高亮，过渡边界时切换自然。

---

### Step 3.3  动作执行引擎

实现所有动作类型：

| 动作类型 | 实现要点 |
|----------|---------|
| 发送快捷键 | 模拟键盘输入（SendInput / keybd_event）|
| 模拟输入文本 | 逐字符模拟，注意输入法状态 |
| 粘贴内容 | 写入剪贴板 + 发送 Ctrl+V |
| 打开路径/URL | ShellExecute，展开环境变量 |
| 运行动作引用 | 通过引用 ID 查找并执行目标动作，失联时弹出提示 |
| 发送文本 | 支持 `FILE:` 前缀从文件读取 |
| 插入日期时间 | 解析 `{0:格式}` 替换符，格式化当前时间后粘贴 |

- 动作引用失联（原始动作已删除）时：显示错误提示，不崩溃

**验收标准**：每种动作类型分别测试，执行结果与预期一致。

---

### Step 3.4  场景上下文切换

- 监听系统前台窗口变化事件（`WinEvent FOREGROUND`）
- 获取当前活动窗口的进程名（不含路径，如 `notepad.exe`）
- 与配置中的软件场景列表匹配：
  - 匹配成功 → 使用该软件场景覆盖全局场景
  - 无匹配 → 使用全局场景
- 实现 F1 重复触发：轮盘展开时按 F1 立即执行当前高亮动作，轮盘不关闭
- 实现超时自动取消：定时器从 WHEEL_SHOWN 开始计时，超时调用关闭流程

**验收标准**：切换到 notepad 后轮盘动作变为 notepad 场景配置，切走后恢复全局。

---

## Phase 4  集成 & 测试

**目标**：三层打通，全场景验收。

### Step 4.1  三层对接

- Phase 3 触发时读取 Phase 2 配置 → 传给 Phase 1 渲染正确动作内容
- Phase 2 设置修改后实时生效（或重新加载配置）
- 确认数据流：`设置配置文件 → 功能逻辑读取 → 渲染层显示`

### Step 4.2  边界 & 异常测试

- 多显示器环境：圆盘在各显示器上定位正确
- 高 DPI（125%/150%/200%）：尺寸等比缩放，不模糊
- 屏幕四角触发：边缘限制模式正确工作
- 动作引用失联：提示错误，不崩溃
- 触发键冲突：已设置为触发键的按键在其他功能中有冲突时给出警告

### Step 4.3  主题 & 外观验证

- 浅色 / 深色主题切换，颜色变量正确应用
- 自定义外观皮肤（第三方主题）兼容性验证

### Step 4.4  性能优化

- 鼠标事件响应延迟目标：< 16 ms（一帧内）
- 动画帧率：稳定 60 fps
- 内存无泄漏（长时间运行测试）
- 钩子卸载：进程退出时正确移除全局鼠标钩子

## 阶段依赖关系总览

```
Phase 1 ──────────────────────────────────────────┐
  （可独立运行，假数据填充）                          │
                                                   ↓
Phase 2 ──────────────────────────────────────────┤
  （可独立运行，读写配置文件）                        │
                                                   ↓
Phase 3 ←───── 依赖 Phase1 渲染层 ←────── 依赖 Phase2 配置层
  （实现真实触发 + 执行逻辑）
                                                   ↓
Phase 4
  （三层全部接入，集成验收）
```

## 文件结构建议

```
/src
  /renderer          ← Phase 1：圆盘渲染相关代码
    circle_draw.py      圆盘几何绘制
    sector_highlight.py 扇区高亮状态
    animation.py        弹出/关闭动画
  /settings          ← Phase 2：设置界面代码
    config_ui.py        设置面板 UI
    scene_manager.py    场景管理
    config_store.py     本地配置读写
  /logic             ← Phase 3：功能逻辑代码
    mouse_hook.py       全局鼠标钩子
    state_machine.py    触发状态机
    direction.py        方向判断算法
    action_engine.py    动作执行引擎
    context_switch.py   场景上下文切换
  /tests             ← Phase 4：测试代码
    test_renderer.py
    test_config.py
    test_logic.py
    test_integration.py
  main.py            ← 入口，组装三层
```
