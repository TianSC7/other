## 一、文件位置与命名

| 项目 | 值 |
|------|----|
| 文件路径 | `%APPDATA%\WheelMenu\config.json` |
| 展开示例 | `C:\Users\张三\AppData\Roaming\WheelMenu\config.json` |
| 备份路径 | `config.json.bak.yyyyMMddHHmmss`（同目录）|
| 临时写入路径 | `config.json.tmp`（原子写入用，写完后 Move 覆盖）|
| 编码 | UTF-8 with BOM（兼容性最佳）|
| 最大文件大小 | 软限制 1MB（超出时警告，不阻止保存）|

## 四、各动作类型 value 字段规范

| type | value 格式 | 示例 |
|------|-----------|------|
| `none` | 忽略 | — |
| `hotkey` | `修饰键+键名`，多个修饰键用 `+` 连接 | `Ctrl+Shift+S`、`Alt+F4`、`F5` |
| `simulateInput` | 普通文字直接写；按键用 `{键名}` | `{ctrl}{a}Hello{enter}` |
| `paste` | 任意 Unicode 文本 | `你好世界` |
| `open` | 文件路径 / 文件夹路径 / URL，支持环境变量 | `%APPDATA%\WheelMenu`、`https://google.com` |
| `runAction` | 忽略（使用 `actionRefId`）| — |
| `sendText` | 文本，或 `FILE:绝对路径` | `FILE:C:\snippets\sig.txt` |
| `dateTime` | `string.Format` 格式字符串 | `{0:yyyy年MM月dd日 HH:mm}` |

### hotkey 支持的键名（大小写不敏感）

```
修饰键：Ctrl / Control、Alt、Shift、Win
功能键：F1 ~ F12
导航键：Enter、Escape（Esc）、Tab、Space、Backspace（Back）
         Delete（Del）、Insert（Ins）、Home、End、PageUp、PageDown
方向键：Up、Down、Left、Right
数字键：0~9（键盘数字行，非小键盘）
字母键：A~Z（单字符）
小键盘：NumPad0~NumPad9、Multiply、Add、Subtract、Divide、Decimal
```

### simulateInput 支持的 {键名} 列表（同 hotkey 键名）

```
示例：
  {ctrl}{a}           → 全选
  {ctrl}{c}           → 复制
  Hello{enter}        → 输入 "Hello" 并回车
  {shift}{home}       → Shift+Home 选中到行首
```

## 五、方向键命名规范

### 8 格模式方向名

```
         N (0°)
    NW       NE
  (315°)   (45°)

W (270°)     E (90°)

    SW       SE
  (225°)   (135°)
         S (180°)
```

### 16 格模式方向名（外圈专用）

```
N(0°) NNE(22.5°) NE(45°) ENE(67.5°) E(90°) ESE(112.5°) SE(135°) SSE(157.5°)
S(180°) SSW(202.5°) SW(225°) WSW(247.5°) W(270°) WNW(292.5°) NW(315°) NNW(337.5°)
```

> 内圈和扩展圈始终只使用 8 格方向名（N/NE/E/SE/S/SW/W/NW），`outerRing16Mode=true` 时外圈方向名扩展为 16 个。
> 存储时使用完整方向名字符串（不使用数字索引），便于跨设备同步时理解配置含义。

## 六、软件场景覆盖语义

```
解析规则（代码参考 ConfigService.ResolveSectorAction）：

IF 当前进程名 存在于 scenes 字典 AND 该场景的对应圈方向 有值（包括 null）:
    使用软件场景的值
    （null = 显式清空，即"此位置无动作，不回退全局"）
ELSE:
    使用 global 场景的值
    （global 缺省 = 无动作）
```

### 示例对比

```
全局 innerRing.N = Ctrl+C
notepad innerRing.N = Ctrl+S     → notepad 下北方向 = Ctrl+S
notepad innerRing.N = null       → notepad 下北方向 = 无动作（不显示全局的 Ctrl+C）
notepad innerRing 中无 N 键      → notepad 下北方向 = 全局的 Ctrl+C（回退）
```

## 七、版本迁移策略

### 版本号规则

- 配置文件根节点有 `version` 整数字段
- 每次**不向后兼容的结构变更**时版本号 +1
- 纯新增字段（有默认值）不需要升级版本号，`System.Text.Json` 的 `MissingMemberHandling` 默认忽略缺失字段

### 当前版本历史

| 版本 | 变更说明 |
|------|---------|
| v1（当前）| 初始格式 |

### 迁移实现模板

```csharp
// ConfigService.cs 中的 MigrateIfNeeded
private static void MigrateIfNeeded(WheelConfig config)
{
    if (config.Version == CurrentVersion) return;

    // 每个迁移步骤按顺序执行，确保 v1→v3 经过 v1→v2→v3
    if (config.Version < 2) MigrateV1ToV2(config);
    if (config.Version < 3) MigrateV2ToV3(config);
    // ...

    // 迁移完成后立即保存（防止迁移后崩溃导致每次重新迁移）
    // 注意：不能在 Load() 中直接调用 Save()，应返回后由调用方处理
    config.Version = CurrentVersion;
}

private const int CurrentVersion = 1;

// 示例：v1 → v2（假设 v2 将 triggerKey 从字符串改为枚举数字）
private static void MigrateV1ToV2(WheelConfig config)
{
    // 读取旧字段，写入新字段，删除旧字段
    // ...
    config.Version = 2;
}
```

### 迁移时的备份策略

```
迁移执行前：自动创建备份（config.json.bak.v{旧版本}.{时间戳}）
迁移成功后：立即保存新版本配置
迁移失败：使用默认配置，提示用户备份位置
```

## 八、原子写入实现

防止写入中途崩溃导致配置文件损坏：

```csharp
public void Save(WheelConfig config)
{
    Directory.CreateDirectory(Path.GetDirectoryName(ConfigPath)!);

    // 序列化到内存
    var json = JsonSerializer.Serialize(config, JsonOpts);

    // 写入临时文件
    var tmpPath = ConfigPath + ".tmp";
    File.WriteAllText(tmpPath, json, System.Text.Encoding.UTF8);

    // 原子替换（同分区内 Move 是原子操作）
    File.Move(tmpPath, ConfigPath, overwrite: true);

    _cache = config;
    ConfigSaved?.Invoke(config);
}
```

> **为什么是原子的**：在同一磁盘分区内，`File.Move` 在 NTFS 上是原子操作（重命名目录项），不会出现"写了一半"的状态。临时文件写失败不影响原文件。

## 九、图标路径规范

| 支持格式 | 加载方式 |
|---------|---------|
| `.png` | `BitmapImage` 直接加载 |
| `.ico` | `System.Drawing.Icon` 提取，转换为 `BitmapSource` |
| `.svg` | 暂不支持（Phase 1 版本），后续可用 SharpVectors 库 |
| EXE 内嵌图标 | `System.Drawing.Icon.ExtractAssociatedIcon(path)` |
| `null` | 不显示图标，只显示文字标签 |

图标路径存储规则：
- 优先存储**绝对路径**（避免相对路径在工作目录变化时失效）
- 图标文件建议统一复制到 `%APPDATA%\WheelMenu\icons\` 目录下
- 路径不存在时：降级为只显示文字，不报错

## 十、常见错误配置及处理

| 错误情况 | 处理方式 |
|---------|---------|
| JSON 格式非法 | 备份原文件，使用默认配置，弹出提示 |
| `version` 字段缺失 | 视为 v1 处理 |
| `scenes` 中无 `global` 键 | 自动插入空全局场景 |
| `type` 为未知枚举值 | 该格子降级为 `none`，其余格子正常加载 |
| `iconPath` 指向不存在文件 | 忽略图标，使用文字显示 |
| `triggerKey` 为未知值 | 回退默认值 `"middle"` |
| `size` 超出范围 | 截断到 [50, 300] |
| `timeoutMs` 为负数 | 视为 0（不超时）|
| `actionRefId` 为空字符串 | 执行时弹出失联提示 |

```csharp
// 安全读取示例（防御性编程）
private static WheelConfig SafeDeserialize(string json)
{
    var config = JsonSerializer.Deserialize<WheelConfig>(json, JsonOpts) ?? new();

    // 确保 global 场景存在
    if (!config.Scenes.ContainsKey("global"))
        config.Scenes["global"] = new SceneConfig { Name = "全局" };

    // 截断异常值
    config.Settings.Size      = Math.Clamp(config.Settings.Size, 50, 300);
    config.Settings.TimeoutMs = Math.Max(0, config.Settings.TimeoutMs);

    return config;
}
```
