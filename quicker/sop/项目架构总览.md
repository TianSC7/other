## 三、层级依赖规则（严格单向）

```
┌──────────────────────────────────────────────────────┐
│  App.xaml.cs（组装层）                                 │
│  唯一允许跨层引用的地方                                 │
└──────────┬───────────────────────────────────────────┘
           │ 依赖
    ┌──────┴──────┬──────────────┬──────────────┐
    ▼             ▼              ▼              ▼
Renderer/    Settings/       Logic/        Windows/
（渲染层）    （配置层）       （逻辑层）     （窗口层）

允许的依赖方向：
  Logic/     → Settings/Services  (读配置)
  Logic/     → Renderer/          (调用渲染接口)
  Logic/     → Win32/             (P/Invoke)
  Logic/     → Windows/           (控制窗口显示)
  Settings/  → 无其他层依赖（仅依赖 Models 和自身 Services）
  Renderer/  → 无其他层依赖（纯绘制）
  Windows/   → Renderer/          (宿主 WheelCanvas)

严格禁止的依赖方向：
  ❌ Renderer/ → Logic/
  ❌ Renderer/ → Settings/
  ❌ Settings/ → Logic/
  ❌ Settings/ → Renderer/（WheelPreviewCanvas 例外，仅复用 WheelGeometry/WheelConstants）
```

> **Helpers/** 是工具类，可被任意层引用，但本身不依赖任何层。
> **NativeMethods.cs** 是 P/Invoke 声明，`internal` 访问级别，仅 Logic 层使用。

## 四、关键类职责一览

### Renderer 层

| 类 | 职责 | 公开接口 |
|----|------|---------|
| `WheelConstants` | 所有尺寸、颜色、动画常量（`static`）| 只读常量字段 |
| `WheelGeometry` | 扇形路径计算、坐标转换、HitTest（`static`）| `CreateSectorRing()`、`HitTest()`、`SectorCenterPoint()` |
| `WheelSectorData` | 单格子显示数据（POCO） | `HasAction`、`Label`、`Icon`、`GetDisplayMode()` |
| `WheelCanvas` | 核心绘制控件，继承 `FrameworkElement` | `SetCenter()`、`SetHighlight()`、`ClearHighlight()`、`SetDisplayOptions()`、`InnerData[]`、`OuterData[]` |
| `WheelAnimator` | 弹出/关闭动画封装 | `Open(Point, Action?)`、`Close(Action?)` |
| `FakeData` | Phase 1 假数据（Phase 3 后删除） | `InnerRing[]`、`OuterRing[]` |

### Settings 层

| 类 | 职责 | 说明 |
|----|------|------|
| `WheelConfig` | JSON 根节点（POCO） | 含 `Version`、`Settings`、`Scenes` |
| `ConfigService` | 单例，配置读写、事件广播 | `Load()`、`Save()`、`ConfigSaved` 事件、`ResolveSectorAction()` |
| `SettingsWindowViewModel` | 设置窗口主 VM，管理 dirty 状态 | `Save()`、`Cancel()`、`IsDirty`、`CanClose()` |
| `WheelPreviewCanvas` | 设置界面圆盘示意图（静态，可点击）| `SectorClicked` 事件、`CurrentScene`、`GlobalScene` |
| `SectorEditViewModel` | 格子编辑弹窗 VM，7 种动作类型 | `ToConfig()`、`LoadFrom()`、`IsHotkeyType` 等布尔属性 |

### Logic 层

| 类 | 职责 | 说明 |
|----|------|------|
| `NativeMethods` | 所有 Win32 P/Invoke（`internal static`）| 集中管理，不对外暴露 |
| `LowLevelMouseHook` | 全局低级鼠标钩子封装 | `Install()`、`Uninstall()`、`MouseEvent` 事件 |
| `WheelStateMachine` | 核心协调器，状态转换 + 驱动渲染 | `RepeatTrigger()`、实现 `IDisposable` |
| `ActionExecutor` | 动作执行分派器 | `Execute(SectorActionConfig)` |
| `ForegroundWatcher` | WinEvent 前台窗口监听 | `CurrentProcessName`、`ForegroundProcessChanged` 事件 |
| `SceneResolver` | 当前有效场景解析 + 构建渲染数据 | `BuildSectorDataArray()`、`CurrentProcessName` |

### Windows 层

| 类 | 职责 | 公开接口 |
|----|------|---------|
| `WheelWindow` | 透明全屏宿主窗口 | `ShowWheel(Point)`、`HideWheel()`、`EnableMousePassThrough(bool)`、`RepeatTriggerRequested` 事件 |

## 七、App.xaml.cs 组装顺序

```csharp
// 正确的初始化顺序（依赖必须在被依赖者之前创建）
1. WheelWindow       wheelWindow  = new();   // 宿主窗口
2. WheelCanvas       canvas       = wheelWindow.WheelCanvas;
3. WheelAnimator     animator     = new(canvas, wheelWindow);
4. ConfigService     configSvc    = ConfigService.Instance;   // 单例
5. ForegroundWatcher fgWatcher    = new();
6. SceneResolver     resolver     = new(fgWatcher, configSvc);
7. ActionExecutor    executor     = new();
8. LowLevelMouseHook hook         = new();
9. WheelStateMachine stateMachine = new(hook, animator, canvas, wheelWindow,
                                        executor, resolver, configSvc);
10. fgWatcher.Start();
11. hook.Install();    ← 最后安装钩子，确保其他组件已就绪
```

> ⚠️ `hook.Install()` 必须**最后执行**。钩子安装后立即开始接收鼠标事件，若此时 `stateMachine` 未初始化完成会引发空引用。

## 八、线程模型

| 代码位置 | 运行线程 | 注意 |
|---------|---------|------|
| `LowLevelMouseHook.HookCallback` | **非 UI 线程**（系统钩子线程）| ⚠️ 所有 UI 操作必须通过 `Dispatcher.Invoke` |
| `ForegroundWatcher.OnWinEvent` | **非 UI 线程** | 同上 |
| `WheelStateMachine.OnMouseEvent` | **非 UI 线程** | 调用 `canvas.*` 前必须 `Dispatcher.Invoke` |
| `ActionExecutor.Execute` | 在 `Task.Delay(50).ContinueWith` 中，**线程池线程** | 执行 `SendInput` 等 Win32 API 无需 UI 线程 |
| `ConfigService.Save/Load` | 任意线程（内部无锁，Phase 2 仅在 UI 线程调用） | Phase 3 若需在钩子线程读取，需加锁或用 `Volatile` |
| `WheelCanvas.OnRender` | **UI 线程**（WPF 渲染线程调用）| 不可阻塞 |

**线程安全模板：**
```csharp
// 钩子线程中更新 UI
Application.Current.Dispatcher.Invoke(() =>
{
    _canvas.SetHighlight(ring, sector);
});

// 或非阻塞版本（不等待完成）
Application.Current.Dispatcher.BeginInvoke(() =>
{
    _canvas.SetHighlight(ring, sector);
});
```

## 十一、已知设计决策记录（ADR）

| 决策 | 选择 | 放弃的方案 | 原因 |
|------|------|----------|------|
| 渲染方式 | `FrameworkElement` + `DrawingContext.OnRender` | XAML + Path 元素 | 减少 Visual 树对象数量，高亮切换时只需 `InvalidateVisual()` 而不重建元素 |
| 动画实现 | WPF `Storyboard` + `DoubleAnimation` | `CompositionTarget.Rendering` 手动帧 | 代码量更少，WPF 自动在 UI 线程调度，无需手动帧计时器 |
| 配置序列化 | `System.Text.Json` | `Newtonsoft.Json` | .NET 8 内置，无需额外 NuGet，AOT 兼容 |
| ViewModel 框架 | `CommunityToolkit.Mvvm` | 手写 INotifyPropertyChanged | Source Generator 减少样板代码，`[ObservableProperty]` 自动生成属性 |
| 鼠标穿透策略 | 动态切换 `WS_EX_TRANSPARENT` | `SetWindowRgn` 圆形区域 | `SetWindowRgn` 影响阴影和动画；动态切换更灵活，圆盘显示期间取消穿透即可 |
| 配置写入 | 原子写入（.tmp → Move）| 直接 `File.WriteAllText` | 防止写入中途崩溃导致配置损坏 |
| 进程名匹配 | 仅进程名（小写，含 .exe）| 完整 EXE 路径 | 路径因安装位置不同而变化；进程名稳定，且 Quicker 原版也是此方案 |
